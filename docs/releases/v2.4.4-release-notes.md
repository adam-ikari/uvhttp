# UVHTTP v2.4.4 Release Notes

**Release Date**: 2026-02-26
**Version**: 2.4.4
**Type**: Patch Release (Critical Bug Fix)

## Overview

UVHTTP v2.4.4 is a **critical bug fix release** that addresses a severe memory leak issue in connection cleanup. This release is strongly recommended for all users, especially those running long-running production servers.

## 🚨 Critical Fixes

### Memory Leak in Connection Cleanup

**Severity**: 🔴 High
**Impact**: Production server crashes due to memory exhaustion

**Problem**:
- `uvhttp_connection_free()` was leaking 1,932,392 bytes per connection
- Resources (read_buffer, request, response, ssl) were never freed
- The function set `conn->freed = 1` before async close completed
- This prevented the second call (from `on_handle_close`) from freeing resources

**Solution**:
- Created new internal function `uvhttp_connection_free_resources()`
- Removed `conn->freed = 1` from first call to allow cleanup
- Updated `on_handle_close` to call `uvhttp_connection_free_resources()`
- Resources now properly freed when all handles are closed (`close_pending == 0`)

**Verification**:
- ✅ Valgrind: "All heap blocks were freed -- no leaks are possible"
- ✅ Before: 1,932,392 bytes leak per connection
- ✅ After: 0 bytes leak
- ✅ All 39 unit tests pass
- ✅ All integration tests pass

## 🔄 Migration

### No Breaking Changes

This is a patch release with full backward compatibility. No API changes required.

### Upgrade Instructions

```bash
# Option 1: Git pull (if tracking release branch)
git checkout release
git pull origin release

# Option 2: Download release
wget https://github.com/adam-ikari/uvhttp/releases/download/v2.4.4/uvhttp-2.4.4.tar.gz
tar -xzf uvhttp-2.4.4.tar.gz
cd uvhttp-2.4.4

# Build
mkdir build && cd build
cmake ..
make -j$(nproc)

# Run tests
./uvhttp_unit_tests
```
